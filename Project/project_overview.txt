This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

AI-powered educational platform (Bachelor's thesis). Two components:

1. **FullstackApp** ‚Äî Main web app: Django REST API + React/Vite frontend
2. **Functionalities** ‚Äî Standalone utilities: PDF extraction, TTS narration, RAG pipeline
Teachers create classes, post assignments, upload PDFs. Students enroll, submit work, generate AI Manim animations, and take AI-generated quizzes.

## FullstackApp ‚Äî Backend (Django)

**Location:** `FullstackApp/Backend/`
**Python venv:** `FullstackApp/Backend/venv/`

### Architecture

- `core/` ‚Äî Django project settings + URL root (`/api/` ‚Üí `api.urls`, `/admin/`)
- `api/models.py` ‚Äî All models
- `api/auth_views.py` ‚Äî JWT auth: signup, login, me, token refresh
- `api/views.py` ‚Äî All business logic as `@api_view` functions (no class-based views)
- `api/serializers.py` ‚Äî DRF serializers (student-facing vs teacher-facing variants)
- `api/urls.py` ‚Äî All routes under `/api/`
- `api/ai_pipeline.py` ‚Äî AICourse background pipeline + RAG queries (hardcoded `GEMINI_API_KEY`, `DEEPGRAM_API_KEY`)
- `api/manim_pipeline.py` ‚Äî ConceptAnimation pipeline: Gemini ‚Üí Manim code ‚Üí mp4
- `api/quiz_pipeline.py` ‚Äî Quiz generation pipeline: narration text ‚Üí Gemini ‚Üí 20 MCQs (hardcoded `GEMINI_API_KEY`)
- `api/apps.py` ‚Äî `ApiConfig.ready()` resumes any `PROCESSING` AICourses on startup

**Auth:** JWT via `djangorestframework-simplejwt`. Access tokens: 5 min; refresh tokens: 1 day. Token payload includes `role` and `firstName`. Frontend sends `Authorization: Bearer <token>`.

**Role separation:** Student endpoints: `/api/classes/`, `/api/assignments/`. Teacher endpoints: `/api/teacher/`. Views check `request.user` ownership manually ‚Äî no DRF permission classes beyond `IsAuthenticated`.

**File uploads:** `Backend/media/` ‚Äî submissions ‚Üí `submissions/`, attachments ‚Üí `attachments/`, AI course output ‚Üí `ai_courses/<id>/`, animation scripts/videos ‚Üí `animations/anim_<id>/`.

**CORS:** Only `http://localhost:5173` allowed.

### Models

- `User` ‚Äî email-based auth, `role=student|teacher`
- `Class`, `Enrollment`, `Announcement`, `Assignment`, `AssignmentAttachment`, `Lecture`, `LectureFile`, `Submission`, `SubmissionFile` ‚Äî standard academic models
- `AICourse(class_obj, title, filename, status, current_step, total_pages, error_msg, output_dir)` ‚Äî status: `PROCESSING ‚Üí READY | ERROR`; step: `PENDING ‚Üí EXTRACTING ‚Üí NARRATING ‚Üí TTS ‚Üí CHROMA ‚Üí DONE`
- `ConceptAnimation(ai_course, concept, concept_key, status, video_file, retry_count, error_msg)` ‚Äî status: `PENDING ‚Üí GENERATING ‚Üí RENDERING ‚Üí READY | ERROR`; `unique_together = (ai_course, concept_key)`
- `Quiz(ai_course, status, questions JSONField, error_msg)` ‚Äî `OneToOneField` to `AICourse`; status: `PENDING ‚Üí GENERATING ‚Üí READY | ERROR`; questions is a list of `{question, options[4], correct_index}`
- `QuizAttempt(quiz, student, answers JSONField, score, correct_count)` ‚Äî one attempt per student per quiz

**`Assignment.status`** is intentionally denormalized ‚Äî reflects the last student action across all students, not per-student state. Resubmission resets `graded ‚Üí submitted`; unsubmit resets to `pending`.

## AI Features (Backend Pipelines)

### AICourse Pipeline (`api/ai_pipeline.py`)

Teacher uploads PDF ‚Üí daemon thread runs 4 idempotent steps ‚Üí students browse narrated pages + ask RAG questions.

| Step       | Output                              | Done when                                                                 |
|------------|-------------------------------------|---------------------------------------------------------------------------|
| EXTRACTING | PNGs + JSONs via `ingest_pdf`       | `course_index.json` exists, page count matches PDF, all `page_XXXX.png` present |
| NARRATING  | narration JSONs via `narrate_course`| all `page_XXXX_narration.json` present                                    |
| TTS        | MP3s via `generate_audio_for_course`| all `audio/page_XXXX.mp3` present and valid size                          |
| CHROMA     | chunks added to shared ChromaDB     | `chroma_indexed.marker` exists                                            |

File naming: `page_0001.png`, `page_0001_narration.json`, `audio/page_0001.mp3`.

**RAG:** `query_rag()` uses the shared OSSU ChromaDB at `Functionalities/RAG/chroma_ossu_db/` with `gemini-2.5-flash`. Course content coexists with OSSU content; chunks are prefixed `[Course {id} | Page {n}]`. The ChromaDB must be seeded first by running `Functionalities/RAG/createDB.py` before RAG queries work.

**Resume on restart:** `ApiConfig.ready()` in `apps.py` resumes all `PROCESSING` AICourses. The double RuntimeWarning from StatReloader is expected and harmless.

Console format: `[AI Pipeline] Course {id} ‚Äî Step N: ...`

### ConceptAnimation Pipeline (`api/manim_pipeline.py`)

Student requests an animation ‚Üí daemon thread, up to `MAX_RETRIES = 3`:
1. **GENERATING** ‚Äî Gemini generates Manim code using `prompt_universal`; strips markdown fences
2. Writes `scene.py` to `media/animations/anim_{id}/`
3. **RENDERING** ‚Äî `manim -ql --media_dir <work_dir> scene.py ConceptScene` (subprocess, 180s timeout)
4. Finds `.mp4`, saves path, marks `READY`

Scene class must be named `ConceptScene`. `manim` CLI must be on PATH in the Django venv.

Caching: `concept_key = concept.strip().lower()`. Existing `READY` animation for same `(ai_course, concept_key)` is returned immediately.

Console format: `[Manim Pipeline] Animation {id} ‚Äî ...`

### Quiz Pipeline (`api/quiz_pipeline.py`)

Teacher triggers quiz generation for an AICourse ‚Üí daemon thread:
1. Collects all `page_XXXX_narration.json` text (max 40,000 chars)
2. Calls `gemini-2.5-flash` to produce exactly 20 MCQs as a JSON array
3. Validates structure (4 options, valid `correct_index`), saves to `Quiz.questions`, marks `READY`

`Quiz` is `OneToOneField` to `AICourse`. Students submit attempts via `QuizAttempt`; score is computed server-side. Teacher can view per-student results.

Console format: `[Quiz Pipeline] Quiz {id} ‚Äî ...`

### API Endpoints (all under `/api/`)

**AI Teaching:**
- `POST teacher/classes/<id>/ai-courses/` ‚Äî upload PDF
- `DELETE teacher/classes/<id>/ai-courses/<id>/` ‚Äî delete course
- `GET ai-courses/class/<id>/` ‚Äî list courses
- `GET ai-courses/<id>/status/` ‚Äî poll pipeline status
- `GET ai-courses/<id>/page/<n>/` ‚Äî image + audio URL for page n
- `POST ai-courses/<id>/ask/` ‚Äî RAG question
- `POST ai-courses/<id>/reformulate/` ‚Äî simplified re-explanation

**Animations:**
- `POST ai-courses/<id>/animations/` ‚Äî request animation (returns `animation_id`, `status`, `cached`)
- `GET ai-courses/<id>/animations/list/` ‚Äî library
- `GET animations/<id>/status/` ‚Äî poll; returns `video_url` when READY

**Quiz:**
- `POST teacher/classes/<id>/ai-courses/<id>/generate-quiz/` ‚Äî trigger generation
- `GET teacher/classes/<id>/ai-courses/<id>/quiz-results/` ‚Äî per-student results
- `GET ai-courses/<id>/quiz/status/` ‚Äî poll status
- `GET ai-courses/<id>/quiz/questions/` ‚Äî fetch questions (students)
- `POST ai-courses/<id>/quiz/attempt/` ‚Äî submit answers

### Frontend AI Pages

- `AITeachDashboard.jsx` ‚Äî tabs: **Lecture** (audio player + RAG chat + Animate button) and **Animations** (library with status badges, inline video player)
- Animations poll every 4 seconds; on READY switches to Animations tab
- `TeacherClassDashboard.jsx` ‚Äî drag-zone for PDF upload; collapses to compact button when courses exist

## Cross-Cutting Notes

- **Python 3.11** throughout; each subdirectory has its own venv ‚Äî activate the correct one
- **No shared `requirements.txt` at root** ‚Äî backend deps are in `FullstackApp/Backend/venv/`
- **API keys are hardcoded** in `ai_pipeline.py`, `manim_pipeline.py`, and `quiz_pipeline.py` ‚Äî `GEMINI_API_KEY` and `DEEPGRAM_API_KEY` are not read from `.env`; edit those files directly to update keys
- **Manim rendering** uses `-ql` (low quality, no preview window) server-side; scene class must be `ConceptScene`
- **Media files served from `Backend/media/`** require `Authorization: Bearer <token>` on the request ‚Äî plain `<a href>` links won't work for authenticated users
