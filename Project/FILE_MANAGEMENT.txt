DATABASE STRUCTURE
================================================================================

PRODUCTION: PostgreSQL (configured in core/settings/production.py)

Tables by Domain:
  - Users: authentication_user (email, role, password_hash)
  - Classes: api_class (name, code, teacher_id)
  - Enrollments: api_enrollment (class, student_email, student_name)
  - Announcements: api_announcement (class_id, title, content)
  - Assignments: api_assignment (class_id, title, status, due_date)
  - Assignment Files: api_assignmentattachment, api_assignmentfile
  - Submissions: api_submission (assignment, student_email, status, grade)
  - Submission Files: api_submissionfile
  - Lectures: api_lecture (class_id, title)
  - Lecture Files: api_lecturefile
  - AI Courses: api_aicourse (class_id, title, status, current_step)
  - Animations: api_conceptanimation (ai_course, concept, status)
  - Quizzes: api_quiz (ai_course, questions JSON, status)
  - Quiz Attempts: api_quizattempt (quiz, student, answers, score)

================================================================================
KEY CONCEPTS IN FILE ORGANIZATION
================================================================================

1. SEPARATION OF CONCERNS
   - models/ → data layer
   - views/ → API endpoints
   - services/ → business logic (pipelines)
   - serializers.py → data transformation
   - urls.py → routing

2. ENVIRONMENT-SPECIFIC CONFIG
   - base.py → shared settings
   - local.py → development (debug=True)
   - production.py → production (debug=False)
   - __init__.py → conditional loader

3. PACKAGE STRUCTURE (not flat)
   - models/ is a package (can import any model)
   - views/ is a package (all views must be exported in __init__.py)
   - services/ is a package (import directly by module)

4. MEDIA ORGANIZATION
   - AI courses: media/ai_courses/{course_id}/page_{n}.png
   - Audio: media/ai_courses/{course_id}/audio/page_{n}.mp3
   - Animations: media/animations/anim_{id}/
   - Submissions: media/submissions/{submission_id}/

5. FRONTEND ROUTING
   - /student/* → StudentLayout (with sidebar)
   - /student/ai-teach/* → NO sidebar (special case)
   - /teacher/* → flat routes
   - / → public pages
